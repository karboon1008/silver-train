✦ Architecture

Choose an Approach

How the app is structured affects how fast it's built and how easy it is to extend. Three options — pick the one that fits.

A
Layer-First (Classic React Native)
Code organized by technical layer — screens, components, hooks, services. Simple and predictable for a single developer or small team.
Folder structure
src/
  screens/  — all screen files flat
  components/  — shared UI
  hooks/  — all custom hooks
  services/  — API, maps, AI calls
  store/  — Zustand stores

Pros

  • Easiest to start
  • Familiar structure
  • Low ceremony

Cons

  • screens/ grows unwieldy fast
  • Hard to find feature context
  • Cross-feature coupling risk
B
Feature-First (Slice Architecture)
Each domain (auth, trips, map, explore, ai) is a self-contained folder with its own screens, components, hooks, and store slice. Clear ownership, scales well.
Folder structure
src/
  features/
    auth/  screens · hooks · store
    trips/  screens · hooks · store
    map/  screens · hooks · store
    explore/  screens · hooks · store
    ai/  hooks · service
  shared/  UI components, utils

Pros

  • High cohesion per feature
  • Easy to navigate by domain
  • Good for multi-sprint builds

Cons

  • More upfront folder setup
  • Shared code needs discipline
C
Hybrid Feature + Shared Core Recommended
Features are self-contained slices. A shared core/ layer holds the design system, API client, and utilities. Services (Supabase, Maps, OpenAI) live in services/ and are injected into features via hooks. Clean, testable, scales from MVP to full product.

Folder structure
src/
  features/
    auth/   trips/   map/   explore/
  core/
    components/  — Card, Sheet, Button, Badge
    theme/  — colours, typography, spacing
    hooks/  — useTheme, useOffline
  services/
    supabase.ts  openai.ts  maps.ts  fcm.ts
  store/  — Zustand root + slices
  navigation/  — Expo Router config

Pros

  • Design system lives in one place
  • Services are mockable for tests
  • Easy to add offline support
  • Features stay independent

Cons

  • Slightly more structure upfront
