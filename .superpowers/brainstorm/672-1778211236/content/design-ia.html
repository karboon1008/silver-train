Gesture System
All gestures follow a spatial metaphor: horizontal = navigate between things, vertical = reveal depth. No gesture conflicts with React Native's built-in scroll.
↕
Sheet drag
Drag bottom sheet up to expand itinerary. Down to collapse back to peek. Snap points at 30%, 60%, 95%.
←
Card swipe left
Swipe a trip card left to reveal Delete / Archive actions (destructive red, non-destructive grey).
☰
Long-press drag
Long-press an itinerary stop to lift and reorder. Haptic on pickup, spring settle on drop.
⊕
Pinch map
Native Google Maps pinch-to-zoom. Pins cluster at low zoom, expand at high zoom with label bubbles.
→
Swipe right (back)
iOS-native full-screen swipe back on all push screens. Gesture drives the transition animation.
↔
Day strip scroll
Horizontal scroll on the day pill strip. Snaps per day, smooth inertia, selected pill auto-centres.
Microinteractions
Every state change has a physical response. Animations use spring curves (not easing) to feel tactile, not digital.
Tap card
→
Scale down to 0.97 on press, spring back on release. Shared element transition to detail screen.
Add activity
→
Row inserts with a height spring from 0. Map pin appears with a bounce drop. Light haptic impact.
AI generate
→
Shimmer skeleton cards appear immediately. Items replace skeleton one-by-one with a staggered fade-up (80ms offset per card).
Toggle dark mode
→
Cross-fade over 300ms driven by system preference. No flash, no jump.
Complete day
→
Day pill gets a subtle checkmark overlay. Confetti burst (3 particles, not overwhelming) on trip completion.
Offline mode
→
Thin sage banner slides in from top. Cached content still renders. Network-only features grey out with tooltip.
Offline Strategy
Trips are the primary offline asset. Maps tiles cache on-demand. AI and push notifications require connectivity.
Full offline — Trip itineraries, day plans, saved places
Full offline — Downloaded map tiles (per-trip area)
Cached — Destination info, attraction details
Online only — AI generation, live search, push notifications
Storage strategy: Supabase local cache via @supabase/supabase-js + AsyncStorage for trip JSON snapshots. MMKV for fast key-value (theme, auth tokens). No SQLite needed at MVP.
