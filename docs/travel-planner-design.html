Section 05

Relationships
users→ 1:many →trips  user owns trips
trips→ 1:many →days  trip spans days
days→ 1:many →stops  day has ordered stops
stops→ many:1 →places  stops reference cached Google Place data
stops→ 1:many →transport_segments  each stop has segments to the next stop
Row-Level Security
All tables except places enforce: users can only SELECT / INSERT / UPDATE / DELETE their own rows (matched by auth.uid()). The places table is readable by all authenticated users and writable only by the service role (populated by the server when stops are added).
Section 06
Services, Error Handling & Testing
🛢️
Supabase
Auth + Database + Realtime
Email + Google OAuth. All CRUD on trips/days/stops/places. RLS enforced. Realtime sync across devices for same user. Edge Functions host the OpenAI calls and push notification scheduler.
supabase.auth.signIn()
.from('trips').select()
.upsert()
.subscribe()
functions.invoke()
✦
OpenAI
AI Itinerary Generation + Stop Suggestions
Called server-side via Supabase Edge Function only — API key never on device. GPT-4o with response_format: json_schema streams structured day→stops arrays. "Suggest more" passes existing stops as context.
generate-trip (Edge Fn)
suggest-stops (Edge Fn)
response_format: json_schema
🗺️
Google Maps
Maps · Places · Directions
react-native-maps with Google Maps provider. Places Autocomplete for stop search. Place Details populates the places cache (name, category, thumbnail, hours, rating). Directions API fetches transport durations stored in transport_segments. Places re-fetched if cached_at > 7 days.
Places Autocomplete
Place Details API
Directions API

🔔
Firebase FCM
Push Notification Reminders
Trip-day reminders sent via a Supabase scheduled Edge Function each morning. expo-notifications handles FCM token registration and foreground/background display. Users opt out per trip in Profile → Settings.
Expo.Notifications.getToken()
FCM via Edge Function
💾
MMKV + AsyncStorage
Local Persistence & Offline Cache
MMKV: auth tokens, theme, prefs (synchronous reads). AsyncStorage: JSON trip snapshots written on every online view, read when offline detected via @react-native-community/netinfo.
MMKV.set('token', ...)
AsyncStorage.setItem('trip:id', JSON)
Error Handling
ScenarioHandling
OfflineNo networkSage banner slides in. AsyncStorage snapshot loads trips. Network-only features grey out with tooltip.
APIOpenAI failsInline "Couldn't generate — try again" with retry button. No modal, no crash.
APIMaps unavailableTransport pills show "–". Map hero shows static placeholder. Search shows "unavailable".
AuthToken expiredSupabase auto-refreshes. On failure, quiet redirect to Sign In with "Session expired" notice.
UserRemark save failsOptimistic update shown; on failure, value reverts + red underline + retry tap.
OfflineMap app missingLinking.canOpenURL() checked first; only installed apps shown; browser fallback if none.
